LogPill é um aplicativo de acompanhamento de medicação. A ideia é deliberadamente direta: ajudar uma pessoa a saber quais medicamentos precisa tomar hoje, registrar uma dose rapidamente, consultar seu histórico e entender sua adesão ao longo do tempo.

Não é um aplicativo médico regulado. Ele não diagnostica, não recomenda tratamentos e não substitui um profissional de saúde. É uma ferramenta de lembrete, registro e visibilidade pessoal.

O que importa aqui é o ângulo de produto. LogPill é um app B2C completo, com identidade própria, presença nas lojas iOS e Android, site público, visuais de marketing localizados e uma base técnica próxima do que a oferta AppExpress pode produzir para um lançamento real.

## Por que um showcase AppExpress

Projetos AppExpress não são apenas protótipos ou ferramentas internas. O objetivo é entregar algo utilizável: aplicativo móvel, presença web, banco de dados, autenticação quando necessário, páginas legais, assets para as lojas e deploy.

LogPill mostra esse escopo em um caso B2C. O produto parece simples na superfície, mas reúne vários assuntos concretos:

- uma experiência móvel rápida, usada todos os dias;
- lembretes locais confiáveis;
- um banco de dados embarcado, disponível sem conexão;
- telas de análise legíveis sem virar conselho médico;
- textos e capturas localizados em inglês, francês, espanhol e português;
- um protótipo web/SaaS para visualizar como a sincronização em nuvem poderia funcionar se o produto precisasse ir além do telefone.

## O produto

A tela principal mostra o planejamento do dia, as doses restantes e os próximos lembretes. O usuário pode registrar uma tomada, ignorar uma dose ou consultar o histórico sem passar por um fluxo pesado.

A área de medicamentos guarda as informações úteis: nome, dosagem, forma, frequência, quantidade restante e instruções de tomada. As análises focam em tendências: taxa de adesão, sequência atual, doses tomadas ou perdidas, comparação por medicamento e acompanhamento de recargas.

## A stack técnica

A stack não é exótica, e esse é justamente o ponto. LogPill segue os blocos habituais da AppExpress: React Native com Expo para mobile, Next.js para web e Supabase quando o produto precisa de contas, armazenamento ou sincronização.

| Camada | Escolha | Papel |
|---|---|---|
| Mobile | Expo, React Native, TypeScript | App iOS e Android com uma base de código comum. |
| Navegação | Expo Router | Rotas baseadas em arquivos, tabs e telas empilhadas sem configuração pesada. |
| Dados locais | WatermelonDB, SQLite | Registros de tomadas e medicamentos disponíveis offline. |
| Estado da UI | Zustand | Estado de aplicação leve, sem sobre-arquitetura. |
| Lembretes | Expo Notifications | Notificações locais para doses planejadas. |
| Localização | i18next, react-i18next | Interface e assets adaptados aos idiomas de lançamento. |
| Web | Next.js 16, React 19, TypeScript | Site público ativo e protótipo de dashboard web. |
| Backend web | Supabase, Zod, React Hook Form | Base técnica para autenticação, dados, validação e formulários do dashboard. |

No mobile, o coração da experiência é local-first. Uma tomada deve poder ser registrada imediatamente, mesmo sem conexão. Esse detalhe muda a arquitetura: o banco local não é um cache cosmético, ele vira a fonte operacional do app.

O site oficial público está ativo em [logpill.app](https://logpill.app/). O dashboard web, a sincronização em nuvem e o modo SaaS existem no código e nos mockups da AppExpress como protótipo, mas não estão ativos nem implantados na versão pública do LogPill neste estágio. As capturas abaixo documentam essa direção de produto, não uma funcionalidade disponível hoje.

No código, essa parte web usa Next.js, Supabase, Tailwind, Radix UI, Lucide, Framer Motion, React Hook Form, Zod, Resend e React Email.

## O que LogPill valida para AppExpress

LogPill mostra que um produto B2C não se resume a "fazer um app". Também importam a primeira impressão, as lojas, os idiomas, as permissões do sistema, as páginas de suporte, a política de privacidade, o modelo de dados e o comportamento offline.

É exatamente o tipo de produto em que uma stack pragmática importa mais do que uma arquitetura espetacular. As notificações precisam chegar. As doses precisam ser registradas rápido. Os dados precisam continuar legíveis. As capturas precisam ser coerentes em cada idioma. E o site precisa explicar o produto sem criar confusão médica.
