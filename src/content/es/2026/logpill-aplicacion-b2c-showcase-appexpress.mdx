LogPill es una aplicación de recordatorio y seguimiento de medicación. La idea es deliberadamente directa: ayudar a una persona a saber qué medicamentos debe tomar hoy, registrar una dosis rápidamente, consultar su historial y entender su adherencia con el tiempo.

Lo interesante aquí es el ángulo producto. LogPill es una app B2C completa, con identidad propia, presencia en las tiendas de iOS y Android, sitio público, visuales de marketing localizados y una base técnica cercana a lo que AppExpress puede producir para un lanzamiento real.

## Por qué un showcase AppExpress

Los proyectos AppExpress no son solo prototipos o herramientas internas. El objetivo es entregar algo utilizable: aplicación móvil, presencia web, base de datos, autenticación cuando hace falta, páginas legales, assets para las tiendas y despliegue.

LogPill muestra ese alcance en un caso B2C. El producto parece simple en superficie, pero concentra varios temas concretos:

- una experiencia móvil rápida, pensada para uso diario;
- recordatorios locales fiables;
- una base de datos integrada, disponible sin conexión;
- pantallas de análisis legibles sin convertirse en consejo médico;
- textos y capturas localizados en inglés, francés, español y portugués;
- un prototipo web/SaaS para visualizar cómo podría funcionar la sincronización cloud si el producto necesita ir más allá del teléfono.

## El producto

La vista principal muestra el plan del día, las dosis restantes y los próximos recordatorios. El usuario puede registrar una toma, omitir una dosis o consultar el historial sin pasar por un flujo pesado.

La zona de medicamentos guarda los datos útiles: nombre, dosis, forma, frecuencia, cantidad restante e instrucciones de toma. Los análisis se centran en tendencias: tasa de adherencia, racha actual, dosis tomadas o perdidas, comparación por medicamento y seguimiento de recargas.

## La stack técnica

LogPill se apoya en los bloques habituales de AppExpress: React Native con Expo para móvil, Next.js para web y Supabase cuando el producto necesita cuentas, almacenamiento o sincronización.

| Capa | Elección | Rol |
|---|---|---|
| Móvil | Expo, React Native, TypeScript | App iOS y Android con una base de código común. |
| Navegación | Expo Router | Rutas basadas en archivos, tabs y pantallas apiladas sin configuración pesada. |
| Datos locales | WatermelonDB, SQLite | Tomas y medicamentos accesibles sin conexión. |
| Estado UI | Zustand | Estado de aplicación ligero, sin sobre-arquitectura. |
| Recordatorios | Expo Notifications | Notificaciones locales para dosis planificadas. |
| Localización | i18next, react-i18next | Interfaz y assets adaptados a los idiomas de lanzamiento. |
| Web | Next.js 16, React 19, TypeScript | Sitio público activo y prototipo de dashboard web. |
| Backend web | Supabase, Zod, React Hook Form | Base técnica para autenticación, datos, validación y formularios del dashboard. |

En móvil, el núcleo de la experiencia es local-first. Una toma debe poder registrarse inmediatamente, incluso sin conexión. Ese detalle cambia la arquitectura: la base local no es una caché decorativa, sino la fuente operativa de la app.

El sitio oficial público está activo en [logpill.app](https://logpill.app/). El dashboard web, la sincronización cloud y el modo SaaS existen en el código y en los mockups de AppExpress como prototipo, pero no están activos ni desplegados en la versión pública de LogPill en esta etapa. Las capturas siguientes documentan esa dirección de producto, no una función disponible hoy.

En el código, esta parte web usa Next.js, Supabase, Tailwind, Radix UI, Lucide, Framer Motion, React Hook Form, Zod, Resend y React Email.

## Qué valida LogPill para AppExpress

LogPill muestra que un producto B2C no se limita a "hacer una app". También importan la primera impresión, las tiendas, los idiomas, los permisos del sistema, las páginas de soporte, la política de privacidad y el modelo de datos.
