**LLMVoice** is an iOS technical demo built as a testbed for two things: the new iOS 26 speech recognition APIs (**SpeechAnalyzer / SpeechTranscriber**) and running **open-weight LLMs** quantized in **4 bits**, entirely **on-device** through Apple's **MLX** framework.

The idea is direct: you dictate, the app transcribes, then you can either summarize the transcription or send a prompt straight to the model. Everything runs on the device — no server, no API key, no data leaving the iPhone.

<Callout type="tip" title="New: the first reasoning model">
LLMVoice now ships **Qwen3 (0.6B)**, the first reasoning model in the selection. It can run a thinking phase (`<think>`) before its answer, and a button lets you show or hide that reasoning. Watching a 0.6-billion-parameter model reason locally on a phone is still the most interesting part of the project.
</Callout>

Let's be clear: this compressed, the models stay limited and hallucinate readily. The goal is not production but to actually measure what can run on an iPhone, and how far it holds up. My iPhone 13 still heats up the same — but a 0.6B that reasons locally wasn't really on the table when 1.0 shipped.

## Speech recognition (iOS 26)

Transcription relies on **SpeechAnalyzer** and **SpeechTranscriber**, the APIs introduced with iOS 26. They run on-device and feed the rest of the pipeline directly: the transcription becomes either the material for a summary or the context for a prompt.

## Available models

Besides Apple Intelligence (system model, iPhone 15 Pro and later), five open-weight MLX models can be downloaded on demand:

| Model | Hugging Face ID | Size | Context | Languages |
|---|---|---|---|---|
| Gemma 3 (1B) | `mlx-community/gemma-3-1b-it-4bit` | ~300 MB | 32k | 140+ |
| Gemma 3 QAT (1B) | `mlx-community/gemma-3-1b-it-qat-4bit` | ~733 MB | 32k | 140+ |
| Qwen2.5 (0.5B) | `lmstudio-community/Qwen2.5-0.5B-Instruct-MLX-4bit` | ~150 MB | 32k | 29+ |
| Qwen3 (0.6B) — reasoning | `mlx-community/Qwen3-0.6B-4bit` | ~335 MB | 40k | 100+ |
| Llama 3.2 (1B) | `mlx-community/Llama-3.2-1B-Instruct-4bit` | ~500 MB | 128k | 8 |

Qwen3 is the only one with a reasoning mode. For summarization that mode is disabled to keep outputs short; for a direct prompt, the model can lay out its reasoning, which you can show or hide on the fly.

## Measuring performance

A real-time metrics panel accompanies every generation: current, average and peak throughput (tokens/s), token count, total time, time to first token (TTFT), average latency and a progress estimate. This is what makes the demo genuinely useful: comparing models on the same device, under the same conditions.

## Code and HTML

When a model generates code, LLMVoice applies syntax highlighting (HTML, CSS, JavaScript/TypeScript) and offers a rendered preview of the generated HTML, right inside the app.

## System Requirements

iOS 26.0+ / iPadOS 26.0+

**For Apple Intelligence**

- iPhone 15 Pro or later;
- Apple Intelligence enabled in Settings.

**For local MLX models**

- iPhone 12 or later (A14 Bionic chip or higher);
- iPad Pro 2021 or later, iPad Air 5th generation or later (M1 chip or higher);
- Metal GPU with the required features: `air.simd_sum`, `rmsfloat16` kernel support.

## Storage

App size: about 50 MB. MLX models download on demand, from ~150 MB (Qwen2.5) to ~733 MB (Gemma 3 QAT) — see the table above.
